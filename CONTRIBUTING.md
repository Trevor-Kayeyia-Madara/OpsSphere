## How Can I Contribute?

### Reporting Bugs

* Check the **Issues** tab to see if the bug has already been reported.
* If not, open a new issue. Include a clear title, a description of the problem, and steps to reproduce the error.

### Suggesting Enhancements

* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful and how it fits the project's modular architecture.

### Pull Requests

1. **Fork** the repository and create your branch from `main`.
2. **Install dependencies** using `npm install`.
3. **Draft your changes.** Ensure your code follows the existing style (Node.js best practices).
4. **Test your changes.** Make sure the app starts and the REST endpoints work as expected.
5. **Issue a Pull Request.** Provide a detailed description of your changes.

## Coding Standards

* Use **ESLint** and **Prettier** if possible.
* Keep functions small and modular.
* Ensure all environment variables are documented in the `.env.example` file if you add new ones.
